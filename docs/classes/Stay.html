Stay Class

Defined in: src\stay.js:23
Module: Stay

The Stay XHR System.

Used for requesting page content asynchronously while staying on the same page.

Each request can have a hard timeout to avoid endless loading times that are often deemed to fail anyways.

Parameters:

  • options Object
    • The options.
    • [responseFields] Array optional
      • The content container IDs. These have to be the same as the data fields in the server response.
    • [infix] String optional
      • The special url pattern infix for the asynchronous content requests.
    • [timeoutPost] Number optional
      • Hard timeout for POST. 0 means no timeout. Default is 60000 (ms).
    • [timeoutGet] Number optional
      • Hard timeout for GET. 0 means no timeout. Default is 5000 (ms).
    • [autoUpdate] Boolean optional
      • Whether Stay should automatically update the page content. Defaults to true.

update

(
  • response
)

Defined in src\stay.js:350

Updates the view, the navigation listeners and the history state. Also emits an event to signilize that the page has been loaded.

The update function needs to be called after each navigation in order to unlock the system. This happens by default, but that behaviour can be disabled. It is then the responsibility of the programmer to call stay.update() with the response data provided by the "receive" event.

Parameters:

  • response Object
    • The response to display.
